# 样例任务

本目录包含可直接导入使用的任务模板。在程序里点 **导入任务**（任务卡上的「📤 分享」按钮旁的导入入口），选择对应 .json 文件即可。

## 文件清单

### `login_template_demo.json`
**演示 v1.2.0 登录模板功能**。使用公开测试站 [the-internet.herokuapp.com](https://the-internet.herokuapp.com/login)，无需企业内网。

导入后操作步骤：
1. 双击任务卡的「✏️ 编辑」按钮
2. 切到「🔐 登录」Tab
3. 用户名填 `tomsmith`，密码填 `SuperSecretPassword!`
4. 点「💾 保存凭据到密钥库」
5. 关闭对话框，点任务卡的「▶ 立即执行」
6. 任务首次执行时会自动登录；后续执行会复用浏览器 cookie 跳过登录步骤

观察要点：
- 第一次执行：会看到 `检测到未登录，开始回放登录模板` → 登录成功 → 访问 /secure 页面
- 第二次执行：会看到 `检测到已登录（cookie 复用），跳过登录模板` → 直接访问 /secure

如果想测试"登录态过期"行为，可以删除 `browser_cache/logindemo/` 目录后再跑一次。

## 提示：录制你自己的登录模板

样例只能展示固定的 the-internet 测试站。要给真实 OA / 企业系统配置登录模板，请在新建任务时：
1. 进入「🔐 登录」Tab，勾选「启用登录模板」
2. 点「📹 录制登录流程」打开浏览器
3. 走一遍真实登录流程（包括"OA 登录"按钮点击 → 跳转 → 表单填充 → 提交）
4. 在录制列表中选中密码那一步，点「🔒 替换为 ${password}」
5. 类似地处理用户名步骤
6. 点「💾 保存为登录模板」回到编辑器
7. 点「🎯 拾取」选择一个【登录后才会出现】的元素（如顶部头像、欢迎文字）作为已登录标志
8. 填入账号密码，点「💾 保存凭据到密钥库」
9. 保存任务
